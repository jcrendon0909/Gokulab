# 🤝 Guía de Contribución - GŌKU LAB

¡Gracias por tu interés en contribuir a GŌKU LAB! Este documento proporciona pautas y instrucciones para contribuir.

---

## 📋 Código de Conducta

Todos los contribuyentes deben seguir nuestro Código de Conducta:
- Sé respetuoso con todos
- Acepta la crítica constructiva
- Enfócate en lo mejor para la comunidad
- Sé empático con otros contribuyentes

---

## 🎯 Cómo Contribuir

### Reportar Bugs

1. Verifica si el bug ya está reportado
2. Incluye:
   - Descripción clara del problema
   - Pasos para reproducir
   - Comportamiento esperado vs actual
   - Screenshots (si aplica)
   - Información del sistema

### Sugerir Mejoras

1. Describe la mejora claramente
2. Explica por qué sería útil
3. Lista ejemplos de funcionalidades similares

### Pull Requests

1. Fork el repositorio
2. Crea una rama (`feature/descripcion`)
3. Haz commits descriptivos
4. Pushea a la rama
5. Abre un Pull Request

---

## 📝 Convenciones de Commits

```
feat: agregar nueva funcionalidad
fix: corregir bug
refactor: refactorización de código
docs: cambios en documentación
style: cambios de formato/linting
test: agregar o actualizar tests
chore: cambios en build/dependencies
```

---

## ✅ Checklist para PRs

- [ ] Mi código sigue las convenciones del proyecto
- [ ] He actualizado la documentación
- [ ] He agregado tests (si aplica)
- [ ] Todos los tests pasan
- [ ] Mi código no introduce nuevas advertencias

---

## 🎓 Estilo de Código

### TypeScript/JavaScript
- Usa TypeScript para todo código nuevo
- Sigue ESLint config del proyecto
- Usa comentarios significativos
- Mantén funciones pequeñas y enfocadas

### React
- Usa functional components
- Usa hooks en lugar de class components
- Mantén componentes pequeños y reutilizables
- Usa TypeScript para props

### Tailwind CSS
- Usa clases de utilidad
- Evita CSS personalizado innecesario
- Sigue el mobile-first approach
- Usa dark: para dark mode

---

## 🧪 Testing

- Escribe tests para nuevas funcionalidades
- Mantén cobertura > 80%
- Usa Jest para unit tests
- Usa Cypress para E2E tests
